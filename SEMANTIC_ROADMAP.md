# Quicken semantic roadmap: steps 6–8

Steps 1–5 established the algebra needed by the remaining work: `PageRegion`,
`ClientMessage`, `ServerMessage`, `TreeDiff`, `RenderFailure`, Cadence
`Interpretation`, and the Cadence semantics laws. The next work should preserve
the JSON and plain-Go boundaries while making internal state transitions more
precise.

## 6. Connection and session typestate

Model the transport phases in a new `protocol.gp`:

```go
type Phase enum { AwaitingResume; Active; Closed }
type Connection[p Phase] struct { /* erased wsConn plus session */ }
```

Planned transitions:

- `accept(*wsConn) Connection[AwaitingResume]`
- `resume(Connection[AwaitingResume], ResumeMessage) Result[Connection[Active], ResumeFailure]`
- `dispatch(Connection[Active], EventMessage) Result[Connection[Active], TransportFailure]`
- `close(Connection[p]) Connection[Closed]`

The WebSocket loop becomes a total match over `ClientMessage`; only the resume
transition receives the first frame, and only an active connection can dispatch
events. Long polling should share the same `resume` and `dispatch` transition
functions rather than reimplementing phase checks.

Verification: compile-time negative fixtures for event-before-resume and
double-resume, plus parity tests proving WebSocket and long-poll transitions
produce identical `ServerMessage` values.

## 7. Validated identifiers and tokens

Introduce opaque `RegionID`, `PageName`, and `ResumeToken` types in `identity.gp`.
Constructors return `Result[..., IdentityFailure]`; only the JSON/router boundary
accepts strings. Change `Page.regions`, message variants, `SessionStore`, and
route builders to use the opaque types internally. Preserve public convenience
methods accepting strings by validating once and panicking only in authoring
APIs where invalid input remains a programming error.

Verification: generators for valid identifiers, rejection properties for every
invalid rune, and round-trip laws for path and JSON encodings.

## 8. Selective linear capabilities

Use quantity `1` only for genuinely single-use values:

- an unregistered freshly minted `ResumeToken` consumed by session insertion;
- a `mountedRegion` consumed when it is installed into `LiveSession`;
- a floor finalizer consumed when the closing document tail is written.

Do not make `LiveSession`, active connections, or region state linear: they are
intentionally reused. Keep plain-Go boundary guards generated by Go+ so misuse
from non-Go+ callers fails deterministically.

Verification: compile-time negative fixtures for token reuse and double floor
finalization, runtime boundary tests from plain Go, and existing race tests.

## Order

Implement validated IDs first (smallest boundary), typestate second (uses those
IDs in transitions), and linear capabilities last after the transition graph is
stable. Each step must regenerate committed `_gp.go` artifacts and keep the
current browser, WebSocket, long-poll, no-script, and race suites green.
